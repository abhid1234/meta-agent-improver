The benchmark
02 / 09

The setup.

A 30-task ML hyperparameter advisor with a deterministic verifier.

Inner agent

  • Reads experiment log + training script
  • Proposes next best hyperparameter change
  • 80% baseline accuracy (24/30)
  • Struggles with phase ordering
  • Misreads context directives

Outer loop (meta-agent)

  • Reads inner agent's failure traces
  • Proposes edits to the system prompt
  • Runs 21 evolution cycles
  • No fine-tuning, no gradient updates
  • Deterministic verifier — no LLM judge

What the meta-agent discovered
05 / 09

Four rules. Found independently.

  1. Explore architecture before training dynamics, dynamics before LR schedule. Don't jump phases. Follow the natural order.
  2. When context.md gives an explicit directive, it overrides phase ordering. The experimenter's intent is authoritative.
  3. Always try FINAL_LR_FRAC=0.05 first — never 0.1 on a first trial. A high LR floor prevents sufficient decay.
  4. Exclude the baseline row from the "TRIED" list. It's a starting state, not a tested value.

These are what a senior ML engineer tells a junior one in week one. The meta-agent rediscovered standard practice from failure traces alone.

The lesson
08 / 09

Two different levers.

Prompts
Transfer across model families. The rules describe good reasoning, not a model-specific quirk.
+0.003 val_bpb · consistent across A40/L40S/H100
Architecture
Hardware-conditional. depth=8 was the wrong answer on A40 and the best answer on H100.
+0.046 val_bpb on H100 · 15× larger than the prompt win
Counterintuitive GPU note
L40S ($0.79/hr) ran 1,891 steps. H100 ($2.99/hr) ran 1,699 steps. At sub-50M params, kernel overhead dominates.
